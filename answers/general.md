## <a name="var_vs_let_vs_const"></a>var vs let vs const

`var`:

- Область видимости: **функциональная**.
- Особенности: **undefined**, если попытаемся обратиться к переменной на этапе индексации, иными словами: до того, как переменная была объявлена.

`let`:

- Область видимости: **блочная**.
- Особенности: **ReferenceError**, если попытаемся обратиться к переменной на этапе индексации, иными словами: до того, как переменная была объявлена.

`const`:

- Область видимости: **блочная**.
- Особенности:
  - Получим **ReferenceError**, если попытаемся обратиться к переменной на этапе индексации, иными словами: до того, как переменная была объявлена.
  - Значение не может быть переназначено.

## <a name="equalities"></a>Нестрогое равенство (==) vs Строгое равенство (===)

Нестрогое равенство (==) пытается преобразовать значения разных типов к одному типу (числу). В итоге, например, пустая строка и `false` становится нулем, и они будут равны. Оператор строгого равенства (===) проверяет равенство без приведения типов (в лоб, как они есть).

## <a name="null_vs_undefined"></a>null vs undefined

`null` – литерал и ключевое слово языка, которое представляет собой отсутствие какого-либо объекта или же `null` указывает “в никуда”. `undefined` – отсутствие значения как такового.

Оба не имеют свойств и методов и не способны их иметь. Оба, при использовании в качестве условия приводятся к `false`.

Интересный факт: если использовать нестрогое равно `==`, то они не равны ничему, кроме самих себя, а также при использовании `typeof` с `null` мы получим тип “object”. С одной стороны это поведение признано багом JavaScript, а с другой стороны остается с нами для обратной совместимости языка JavaScript.

## <a name="scope"></a>Область видимости в JavaScript

Часть кода, где переменные, объекты и функции могут быть видимы и доступны для использования. В JavaScript их 3: глобальная, функциональная и блочная.

Если переменной не существует в текущей области видимости, ее поиск производится выше, во внешней по отношению к текущей области видимости. Если и во внешней области видимости переменная отсутствует, ее поиск продолжается вплоть до глобальной области видимости. Если в глобальной области видимости переменная обнаружена, поиск прекращается, если нет — выбрасывается исключение. Поиск осуществляется по ближайшим к текущей областям видимости и останавливается с нахождением переменной. Это называется цепочкой областей видимости [(Scope Chain)](https://bool.dev/blog/detail/scope-chain-v-javascript).

## <a name="hoisting"></a>Поднятие (Hoisting) в JavaScript

**Hoisting** - Процесс “поднятия” переменной или функции в глобальную или функциональную область видимости. Hoisting выполняется в контексте выполнения, а он в свою очередь имеет 2 фазы: фаза компиляции и фаза выполнения.

- **Компиляция**. В этой фазе функциональные выражения и переменные, объявленные с помощью ключевого слова «var», со значением undefined поднимаются в самый верх глобальной (или функциональной) области видимости.

- **Выполнение**. В этой фазе переменным присваиваются значения, а функции вызываются или выполняются.

## <a name="closure"></a>Замыкание (Closure) в JavaScript

Механизм, который имеет доступ к переменным внешней функции из внутренней.

```JavaScript
function outerFunction() {
    let outerVariable = "I'm outside!";

    function innerFunction() {
        console.log(outerVariable);
    }

    return innerFunction;
}

const closure = outerFunction();
closure(); // "I'm outside!"
```

## <a name="call_bind_apply"></a>Разница между `call`, `bind`, `apply`

[Function.prototype.call()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/call) – вызывает функцию с переданным контекстом и передает аргументы в виде отдельных значений.

[Function.prototype.apply()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/apply) – вызывает функцию с переданным контекстом, но передает аргументы в виде массива.

[Function.prototype.bind()](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Function/bind) – создает и возвращает новую функцию с переданным контекстом и переданными аргументами. Новую функцию можно вызвать позже.

## <a name="coping"></a>Поверхностное и глубокое копирование

При поверхностном копировании данные копируются только на один уровень вглубь (хорошо подходит исключительно с примитивами, ибо работает быстрее):

- Spread;
- Object.assing()
- Cериализация и сразу же десериализация, используя объект JSON;

Для глубокого копирования лучше использовать `structuredClone()` функцию или же копировать рекурсивно (можно юзануть библиотеку [lodash](https://www.npmjs.com/package/lodash)).

Почему так? Непримитивные типы данных, такие как массивы и объекты, хранятся по ссылке. Так как копирование происходит только на один уровень вглубь, то при копировании происходит копирование ссылок на старые объекты или массивы.

## <a name="tdz"></a>Temporal Dead Zone (TDZ)

Место между началом области видимости переменной и её инициализацией, когда вылетает ошибка `ReferenceError`.

```JavaScript
{
  // TDZ starts here (at the beginning of this block’s local scope)
  // TDZ continues here
  console.log(bestFood); // returns ReferenceError because TDZ continues here
  // TDZ continues here
  let bestFood = "Vegetable Fried Rice"; // TDZ ends here
  // TDZ does not exist here
}
```

RESTful API — это API, разработанный согласно принципам REST.

## <a name="main_rest_principles"></a>Основные принципы REST?

1. **Разделение на клиента и сервер:** Взаимодействие клиента и сервера осуществляется в виде запросов и ответов [(CRUD)](https://ru.wikipedia.org/wiki/CRUD).

2. **Единый протокол:** Взаимодействие между клиентом и сервером должны осуществляться по единому протоколу - HTTP.

3. **Бесстатусное состояние (Stateless) сервера:** Сервер не хранит никакой информации о прошлых запросах/ответах. Каждый запрос и ответ содержат всю информацию, необходимую для завершения взаимодействия. Бесстатусное взаимодействие снижает нагрузку на сервер, экономит память и повышает производительность.

4. **Многоуровневая система:** Возможны дополнительные серверы между клиентом и сервером API в виде слоев для выполнения различных функций.

5. **Кэшируемость:** Делается это в угоду повышения производительности.

## <a name="rest_vs_graphql"></a>REST vs GraphQL

[GraphQL](https://graphql.org/) - позволяет клиентам запрашивать только те данные, которые им нужны. Клиент определяет структуру и формат данных, которые он хочет получить.

Ключевая разница между REST и GraphQL в том, что REST имеет фиксированный формат запроса и ответа для каждого ресурса, а GraphQL нет.

## <a name="logical_or_assignment"></a>Присваивание логического ИЛИ (||=)

Оператор присваивания логического ИЛИ `(x ||= y)` присваивает значение переменной x только в случае, если её текущее значение ложноподобно.

```JavaScript
let a = null;
let b = 10;

a ||= b; // a примет значение 10, потому что a === null (falsy)
console.log(a); // 10

let c = 5;
c ||= 15; // c останется 5, потому что c уже истинное значение (truthy)
console.log(c); // 5
```

## <a name="logical_and_assignment"></a>Присваивание логического И (&&=)

Оператор присваивания логического И `(x &&= y)` присваивает значение переменной x только в случае, если её текущее значение истинноподобно.

```JavaScript
let a = 10;
let b = 5;

a &&= b; // a станет равным b (5), так как a === 10 (truthy)
console.log(a); // 5

let c = 0; // 0 — falsy
c &&= b; // Присваивание не произойдёт, так как c — falsy
console.log(c); // 0
```

## <a name="group_by"></a>Object.groupBy()

Статический метод `.groupBy()` группирует элементы итерируемого объекта, опираясь на переданную колбэк-функцию. Она должна возвращать ключ группы, в которую нужно положить текущий элемент. `Object.groupBy()` сгруппирует элементы в новый объект.

```JavaScript
  { value: '50% 0.2 12', name: 'oklch' },
const colors = [
  { value: '198, 35, 109', name: 'rgb' },
  { value: '55% 0.2 0', name: 'oklch' }
]

const groupedColors = Object.groupBy(colors, (color, index) =>
  color.name === 'oklch' ? 'easyToUnderstand' : 'hardToUnderstand'
)

console.log(groupedColors)

/*
{
  easyToUnderstand: [
    { value: '50% 0.2 12', name: 'oklch' },
    { value: '55% 0.2 0', name: 'oklch' }
  ],
  hardToUnderstand: [
    { value: '198, 35, 109', name: 'rgb' }
  ]
}
*/
```
